For some reason a Ruby array of BigDecimals was being inserted into a String column in our Portgres DB. With a lot of trying out different things and some help from another software developer and DBA we came up with a solution to parse out the numbers. The goal was to convert a string of `["83000.0", "119000.0"]` to an PG array `{83000.0, 119000.0}`. It Take some regex, but itâ€™s doable.

{% highlight sql %}
string_to_array(regexp_replace(["83000.0", "119000.0"], '[\[\]" ]', '', 'g'), ',')
{% endhighlight %}

{% highlight sql %}
  string_to_array   
--------------------
 {83000.0,119000.0}
(1 row)
{% endhighlight %}

### Breaking apart the parts:

Clean up string and remove square brackets.

{% highlight sql %}
regexp_replace(["83000.0", "119000.0"], '[\[\]" ]', '', 'g')
{% endhighlight %}

Split string into array by comma

{% highlight sql %}
string_to_array('"83000.0", "119000.0"'), ',')
{% endhighlight %}

### Conclusion 

I would'nt recommend this for use in production, but works great for some ad hoc querying.
